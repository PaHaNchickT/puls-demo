# Приложение "Puls CRUD Demo"

Деплой: [https://maximum-auto-site.vercel.app/](https://maximum-auto-site.vercel.app/)

## Описание

- Данный проект представляет собой простое веб-приложение для управления списком пользователей с возможностью создания, чтения, редактирования и удаления записей (CRUD).
- Реализована страница с таблицей пользователей. Таблица содержит колонки с информацией о пользователях, имеется поддержка пагинации, сортировки.
- Реализована возможность добавлять новых пользователей, редактировать и удалять существующих.
- Внешний вид страницы выполнен в стиле оригинального сайта app.puls.world/.
- Код и архитектура соответствует стандартам ES2015+. Соблюдаются принципы DRY, KISS, YAGNI и SOLID.

## Особенности

1. Ключевой особенностью данного приложения является глубо проработанная бизнес логика, связанная с ролями пользователей. В рамках приложение соблюдается строгая иерархия "Администратор → Менеджер → Пользователь". Любому пользователю можно назначить в подчинение другого пользователя. У любого пользователя может быть только один начальник, но - любое количество подчиненных. При этом при любых действиях (смена роли, смена подчиненные и удаление пользователя) подчиненные не должны оставаться без начальника, поэтому была реализован алгоритм автоматического подбора нового начальника. Если начальника подобрать не удается или выявляется попытка нарушить иерархию ролей, то возможность выполнения любого из перечисленных действий блокируется. Также при попытке удалить пользователя, у которого есть начальник, высвечивается предупреждение о том, что необходимо дополнительно уведомить о своем решении начальника.
2. Согласно ТЗ, для реализации приложения использование бэкенда было необязательным. Поэтому в качестве работы с данными была использована библиотека Zustand, а в качестве альтернативного хранилища - Local Storage.
3. Для такого простого приложения архитектурные решения, подход к написанию кода и стек могут выглядеть избыточными, но согласно ТЗ при разработке приложения необходимо было показать, как бы я реализовал описанные задачи в условиях реальной работы. Тем не менее, при разработке был соблюден баланс между моей логикой и библиотечной.

## Технологический стек

При разработке проекта использовались следующие технологии:

- **TypeScript и React** - для разработки;
- **Next.js** - для реализации SSR;
- **Material UI** - в качестве библиотеки компонентов;
- **TailWind** - для стилизации;
- **React Hook Form и Zod** - для работы с формами;
- **Zustand** - для работы с данными.

## Инструкции по запуску проекта локально

1. Установите Node.js
2. "Форкните" данный репозиторий: https://github.com/PaHaNchickT/puls-demo
3. Склонируйте только что созданный репозиторий: `https://github.com/<%your_github_username%>/puls-demo/`
4. Перейдите в папку `puls-demo`
5. Чтобы установить все зависимости, используйте `npm install`
6. Для запуска локального сервера используйте `npm run dev`
7. Проект запущен успешно!

## Руководство по скриптам

- `npm run dev` запускает локальный сервер в режиме разработки;
- `npm run build` экспортирует приложение в статический HTML.
- `npm run start` запускает локальный сервер в режиме продакшена;
- `npm run lint` запускает линтер для проверки файлов;
