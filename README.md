# Приложение "Puls CRUD Demo"

![promo-image](https://github.com/user-attachments/assets/53ea2609-61c7-4b81-9890-c8e7bc95ea7f)

Деплой: [https://puls-demo.vercel.app/](https://puls-demo.vercel.app/)

## Описание

- Данный проект представляет собой простое веб-приложение для управления списком пользователей с возможностью создания, чтения, редактирования и удаления пользователей (CRUD).
- Реализована страница с таблицей пользователей. Таблица содержит колонки с информацией о пользователях, имеется поддержка пагинации, сортировки и фильтрации.
- Реализована возможность добавлять новых пользователей, а также редактировать и удалять существующих.
- Внешний вид страницы выполнен в стиле оригинального сайта app.puls.world/.
- Код и архитектура соответствует стандартам ES2015+. Соблюдены принципы DRY, KISS, YAGNI и SOLID.

## Особенности

1. Ключевой особенностью данного приложения является проработанная бизнес логика, связанная с ролями пользователей. В рамках приложение соблюдается строгая иерархия "Администратор → Менеджер → Пользователь". Любому пользователю можно назначить в подчинение другого пользователя. У любого пользователя может быть только один начальник, но одновременно любое количество подчиненных. При этом после совершения одного из действий (смена роли, смена подчиненных или удаление пользователя) подчиненные не должны остаться без начальника, поэтому был реализован алгоритм автоматического подбора нового начальника. Если начальника подобрать не удается или выявляется попытка нарушить иерархию ролей, то возможность выполнения любого из перечисленных действий блокируется. Также при попытке удалить пользователя, у которого есть начальник, высвечивается предупреждение о том, что необходимо дополнительно уведомить о своем решении начальника.
2. Согласно ТЗ, для реализации приложения использование бэкенда было необязательным. Поэтому в качестве работы с данными была использована библиотека Zustand, а в качестве альтернативного хранилища - Local Storage.
3. Для такого простого приложения архитектурные решения, подход к написанию кода и стек могут показаться избыточными, однако ТЗ гласило, что при разработке приложения необходимо было показать, как бы я реализовал описанные задачи в условиях реальной работы. Тем не менее, при разработке соблюдался баланс между моей логикой и библиотечной.

## Технологический стек

При разработке приложения использовались следующие технологии:

- **TypeScript и React** - для разработки;
- **Next.js** - для реализации SSR;
- **Zustand** - для работы с данными.
- **Tailwind CSS** - для стилизации;
- **Material UI** - в качестве библиотеки компонентов;
- **React Hook Form и Zod** - для работы с формами;

## Инструкции по запуску проекта локально

1. Установите Node.js
2. "Форкните" данный репозиторий: https://github.com/PaHaNchickT/puls-demo
3. Склонируйте только что созданный репозиторий: `https://github.com/<%your_github_username%>/puls-demo/`
4. Перейдите в папку `puls-demo`
5. Чтобы установить все зависимости, используйте `npm install`
6. Для запуска локального сервера используйте `npm run dev`
7. Проект запущен успешно!

## Руководство по скриптам

- `npm run dev` запускает локальный сервер в режиме разработки;
- `npm run build` экспортирует приложение в статический HTML.
- `npm run start` запускает локальный сервер в режиме продакшена;
- `npm run lint` запускает линтер для проверки файлов;
